# Screen Recording on Wayland + Hyprland — Recommendations

## Top Recommendation: `wl-screenrec`

**Best balance of performance, simplicity, and Hyprland compatibility.**

- Modern, lightweight, wlroots-native screen recorder
- Hardware-accelerated encoding via VAAPI (H.264, H.265, AV1)
- Minimal overhead compared to `wf-recorder`
- Available in nixpkgs: `wl-screenrec`
- Typical usage: `wl-screenrec -f output.mp4` (fullscreen) or with `slurp` for region

## If you need a feature-rich GUI

**Use OBS Studio** with `xdg-desktop-portal-hyprland`:
- Full feature set (scenes, overlays, multiple sources, streaming)
- Requires: `obs-studio`, `pipewire`, `wireplumber`, `xdg-desktop-portal-hyprland`, `qt6-wayland`
- Source type: "PipeWire" or "Screen Capture" via portal

## Lightweight CLI alternatives

| Tool | Best for | Notes |
|------|----------|-------|
| `wf-recorder` | Universal wlroots recording | Slightly older, widely documented |
| `gpu-screen-recorder` | GPU-only (NVIDIA/AMD), replay buffer | Like ShadowPlay |
| `wfrc` | Wrapper script around wf-recorder/wl-screenrec | Quick keybind-based recording |

## Hyprland-specific / all-in-one tools

- **hypr-snap** — wf-recorder wrapper with post-recording GUI (trim, GIF, clipboard)
- **HyprCapture** — Hyprland plugin + Qt helper, integrates with gpu-screen-recorder
- **grabit** — All-in-one: screenshot + recording + OCR + upload, nix run-able

## Verification

All tools listed are available in nixpkgs (`nixpkgs#wl-screenrec`, etc.) and can be tested on your system directly.
