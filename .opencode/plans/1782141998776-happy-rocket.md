# Fix PDF Export Aspect Ratio

## Problem

PDF export creates slides that don't utilize the page area as well as the live terminal view.
Current export dimensions `rows: 23` produce a very wide page (2.41:1 aspect ratio) that
doesn't match the screen's 16:9 ratio.

## Fix

Change `export.dimensions.rows` from **23** to **31** in `config.yaml`.

This makes the PDF page dimensions:
- Width: 110 × 10px × 0.605 = **666px** (unchanged)
- Height: **31** × 12px = **372px** (was 276px)
- Aspect ratio: **1.79:1** (nearly exact 16:9, was 2.41:1)

## Files to Modify

- `config.yaml:30` — change `rows: 23` to `rows: 31`

## Verification

1. Run `presenter --config-file config.yaml --theme light --export-pdf code-config-curriculum-light.md`
2. Open the resulting PDF and verify the page aspect ratio looks better on screen.
3. Check that no slide content is cut off (the content that fit in 23 rows still fits in 31).
