# Fix `window_close_confirmation=NeverPrompt` error in `wezterm-floating`

## Root Cause

WezTerm's `--config` flag evaluates the right-hand side of `=` as a **Lua expression**.
In the generated `wezterm-floating` wrapper script, the argument:

```
--config "window_close_confirmation=NeverPrompt"
```

passes the raw string `window_close_confirmation=NeverPrompt` to wezterm. WezTerm parses `NeverPrompt` as a Lua variable reference rather than a string literal. Since no Lua variable named `NeverPrompt` exists, it evaluates to `nil`, causing the error:

```
NeverPrompt evaluated as nil
```

## Fix

**File:** `modules/home-manager/wezterm-override/default.nix` line 23

Wrap the value in Lua string quotes (double quotes, escaped for Nix):

```nix
# Before:
"window_close_confirmation=NeverPrompt"
# After:
"window_close_confirmation=\"NeverPrompt\""
```

This produces a bash argument that includes the double quotes in the string passed to wezterm, e.g. `--config 'window_close_confirmation="NeverPrompt"'`, causing WezTerm to parse `"NeverPrompt"` as a Lua string literal — which is the correct type for this config key.
