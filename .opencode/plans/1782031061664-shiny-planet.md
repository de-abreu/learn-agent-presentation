# Fix: Speaker Notes Not Displaying Correctly

## Problems (both from same root cause)

1. **Agent Harnesses note (lines 71-74) not displayed**: The speaker note on the "Context and motivation" slide is never shown on the listening terminal.

2. **Project description note (lines 50-52) shown on wrong slide**: This note appears on the slide that follows ("Context and motivation") instead of the "Project description" slide where it belongs.

## Root Cause

The Agent Harnesses note at lines 71-74 uses a **multi-line comment format** that differs from all other 28 speaker notes:

```markdown
<!--
speaker_note: |
    Agent Harnesses...
-->
```

All other 28 speaker notes use the single-line format (e.g., line 50):

```markdown
<!--speaker_note: |
Content here...
-->
```

Presenterm's parser expects `<!--speaker_note:` on the **same line** as the opening `<!--`. Line 71 is the **only** `<!--` on its own line in the entire file. The multi-line variant is not recognized by presenterm, so:

- Direct bug: The Agent Harnesses note is silently skipped (never published)
- Indirect bug: This desyncs the listener's slide counter by one, causing the **previous** slide's note (Project description, lines 50-52) to appear on the "Context and motivation" slide

## Fix

Edit `code-config-curriculum.md` lines 71-74 to match the single-line format:

**Change** (lines 71-74):
- Move `<!--` and `speaker_note: |` onto the same line: `<!--speaker_note: |`
- Remove leading indentation from the note content

This single change fixes both reported issues.

## Verification

Run the dual-instance speaker notes setup:
1. Terminal 1: `presenterm code-config-curriculum.md --publish-speaker-notes`
2. Terminal 2: `presenterm code-config-curriculum.md --listen-speaker-notes`
3. Navigate to the Project description slide and confirm the note appears there
4. Navigate to the Context and motivation / Agent Harnesses slide and confirm the note appears there
